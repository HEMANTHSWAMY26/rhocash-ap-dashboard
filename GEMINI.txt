# GEMINI.md

# Agent Instructions

> This file defines how the Gemini agent operates within the Rhocash AP Hiring Intelligence System. It follows the Anti-Gravity 3-layer architecture to ensure reliability and deterministic execution.

This system detects **US companies hiring Accounts Payable roles** and produces a **lead intelligence dashboard** with downloadable datasets for outbound sales.

The agent **does not perform heavy logic itself**. It routes tasks to deterministic Python tools in `execution/`.

---

# System Goal

Continuously identify companies hiring for **Accounts Payable roles in the US** and generate a **lead intelligence dashboard**.

Outputs:

* Google Sheet containing job leads
* Hiring intensity scoring
* Interactive dashboard
* Downloadable lead lists

Target:

```
10,000 job postings per week
```

---

# 3-Layer Architecture

## Layer 1: Directive (What to do)

Directives define tasks.

Stored in:

```
directives/
```

Examples:

```
directives/scrape_jobs.md
directives/process_leads.md
directives/update_google_sheet.md
directives/build_dashboard.md
directives/deploy_dashboard.md
```

Each directive defines:

* goal
* inputs
* tools/scripts
* outputs
* edge cases

The agent **must read directives before executing anything**.

---

## Layer 2: Orchestration (You)

Your role is **decision making and routing**.

Responsibilities:

* Read directives
* Determine correct execution tools
* Run tools in correct order
* Handle failures
* Improve directives when needed

You **do not scrape websites directly**.

Instead you:

```
read directive → call execution script
```

Example:

```
directive: scrape_jobs.md
tool: execution/run_apify_scraper.py
```

---

## Layer 3: Execution (Deterministic Tools)

Execution scripts live in:

```
execution/
```

These scripts handle:

* API calls
* data processing
* Google Sheets
* dashboard updates
* file generation

Scripts must be:

* deterministic
* well commented
* testable
* reusable

Examples:

```
execution/run_apify_scraper.py
execution/process_jobs.py
execution/update_google_sheet.py
execution/calculate_hiring_intensity.py
execution/generate_dashboard_data.py
execution/deploy_streamlit.py
```

---

# Operating Workflow

The agent must follow this **strict order of operations**.

---

# Step 1 — Run Job Scraper

Directive:

```
directives/scrape_jobs.md
```

Execution tool:

```
execution/run_apify_scraper.py
```

Purpose:

Collect Accounts Payable job postings.

Sources:

* Indeed
* LinkedIn
* Monster
* Aston Carter
* JobRight
* AccountingCrossing
* Robert Half

Keywords:

```
Accounts Payable
Accounts Payables
Accounts Payable Clerk
Accounts Payable Specialist
Accounts Payable Analyst
Accounts Payable Manager
Full Cycle Accounts Payable
```

Filter:

```
United States only
```

Output:

```
.tmp/raw_jobs.json
```

---

# Step 2 — Process and Clean Data

Directive:

```
directives/process_leads.md
```

Execution tool:

```
execution/process_jobs.py
```

Tasks:

* parse raw jobs
* remove duplicates
* normalize company names
* extract fields

Fields required:

```
Company Name
Job Title
Job Description
City
State
Country
Posted Date
Job URL
Source
```

Output:

```
.tmp/processed_jobs.csv
```

---

# Step 3 — Calculate Hiring Intensity

Directive:

```
directives/calculate_hiring_intensity.md
```

Execution tool:

```
execution/calculate_hiring_intensity.py
```

Logic:

```
1 job → Low
2-4 jobs → Medium
5+ jobs → High
```

Group by:

```
company
```

Output:

```
.tmp/jobs_with_intensity.csv
```

# Self-Annealing Process

If something fails:

1. Read error message
2. Fix execution script
3. Test tool again
4. Update directive with learnings
5. Continue workflow

Do not silently fail.
